CSS

  • CSS

  • O site de referência é o W3Schools
  • Nome oficial: Cascading Style sheets, ou "folhas de estilo em cascata"

    • Prós

    • Suporte a acessibilidade: Uma vez que essa linguagem ajuda muito na capacidade de alterar coisas no site e torna-lo mais acessível, como o daltonismo por exemplo.

    • Apresentação: Com o CSS, é possível completar o HTML com CSS para estilizar e deixar o site mais agradável

    • Controles Precisos: Com o CSS, é mais preciso de se estilizar um ou mais itens de HTML.

    • Contras

    • Sobrescrita: por ser um método de estilos em cascata, pode haver resultados inesperados em códigos abaixo do que está sendo alterado, podendo levar a resultados inesperados

    • Diferença entre navegadores: Alguns navegadores "lêem" o CSS de formas diferentes, oque pode gerar conflitos na programação

    • manutenção complexa: uma vez que quanto maior e mais complexo o site, mais para estilizar o site, já que quanto mais estilos mais difícil é para manter o controle

      • Projetos que usam CSS: Alguns dos projetos são O GitHub, o Google, o Adobe, o Twitter, o Spotify e o G1
